# Critical Points
- [ ] CP1: App loads successfully at local web UI and reaches the chat workspace without console/runtime crash.
- [ ] CP2: Sidebar/project rail opens/collapses/hover-previews smoothly and exposes project/chat navigation.
- [ ] CP3: Switching between chats/projects/spaces does not trigger visible blank/frozen UI or excessive lag.
- [ ] CP4: Chat send flow works: compose, send, optimistic/user message appears, assistant/run status begins, tool/activity surfaces do not corrupt timeline.
- [ ] CP5: Chat history/scroll behavior works: scroll away, jump/latest behavior, older-history prefetch does not show noisy loading text or jump badly.
- [ ] CP6: Activity/inspector surfaces open and remain usable while switching back to chat/sidebar.
- [ ] CP7: Power-user rapid interactions for ~15 minutes surface no major JS console errors, request storms, or UI lockups.
